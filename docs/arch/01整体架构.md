# acore-ranking 整体架构设计

## 一、系统概述

acore-ranking 是一个魔兽世界私服（AzerothCore）数据展示服务，通过直连私服数据库，为玩家提供在线统计、多维排行、硬核模式挑战记录、最近成就、封禁列表等可视化数据面板。所有页面支持通过 `<iframe>` 嵌入到 WordPress 等外部站点，并适配移动端浏览。

### 1.1 设计目标

| 目标 | 说明 |
|-----|------|
| 技术现代化 | 抛弃 legacy PHP 技术栈，采用 Express + React + TypeScript 全栈方案 |
| 工程标准化 | 项目目录、代码结构、开发流程符合行业最佳实践 |
| 云原生部署 | 支持腾讯云云函数（SCF）/ 容器镜像快速部署 |
| 配置外置化 | 所有敏感信息通过 `.env` 环境变量注入，无硬编码 |
| 可嵌入性 | 页面支持 iframe 嵌入，响应头适配跨域场景 |
| 移动适配 | 响应式布局，表格支持横向滑动或卡片式展示 |
| 开源友好 | MIT 许可证，标准化开源仓库管理 |

## 二、技术选型

### 2.1 技术栈总览

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **前端** | React 18.3+ | UI 组件库 |
| | TypeScript 5.3+ | 类型安全 |
| | Vite 5.0+ | 快速构建工具 |
| | React Router 6.20+ | 路由管理 |
| | shadcn/ui + Tailwind CSS 3.4+ | 高质量 UI 组件 + 原子化 CSS |
| | TanStack Table 8.0+ | 现代化表格组件（支持移动端） |
| **后端** | Node.js 20+ | 运行环境 |
| | Express 4.21+ | 轻量灵活的 Node.js Web 框架 |
| | TypeORM 0.3+ | ORM，已验证与 AzerothCore 兼容 |
| | cors + helmet | CORS 跨域、安全响应头 |
| | express-validator | 请求参数校验 |
| | ioredis 5+ | Redis 客户端 |
| **数据存储** | MySQL 8+ | AzerothCore 数据库（只读连接） |
| | Redis 7+ | 热点数据缓存 |
| **部署运维** | Docker + Docker Compose | 容器化部署 |
| | 腾讯云 SCF Web 容器 / TCR | 云函数部署 |
| **日志** | pino | 高性能结构化日志，分级输出 |

### 2.2 选型理由

**为什么选 Express？**
- 轻量灵活，云函数冷启动快，适合本项目规模（14 个 API 端点）
- 生态最成熟，团队学习成本低，出问题容易找到解决方案
- 配合 TypeORM 已验证与 AzerothCore 兼容（acore-api 使用相同 ORM）
- 分层架构（routes / services / repositories / entities）完全由代码组织实现，无框架约束
- 支持多种部署模式（传统服务器 / Serverless / 容器）

**为什么选 React + Vite？**
- 现代化组件化开发，生态成熟
- Vite 构建速度快，开发体验好
- 与 Tailwind CSS 配合实现响应式布局和移动端适配
- 构建产物轻量，适合 iframe 嵌入场景

## 三、系统架构图

### 3.1 全局架构（单体容器部署）

前后端构建为**单一 Docker 镜像**，Express 同时托管 API 服务和前端静态文件：

```
  ┌─────────────────────────────────────────────────────────────────────┐
  │  浏览器用户 / WordPress iframe                                        │
  │  (PC + 移动端)                                                        │
  └─────────────────────────┬───────────────────────────────────────────┘
                            │ HTTPS
                            ▼
  ┌─────────────────────────────────────────────────────────────────────┐
  │  腾讯云 SCF Web 容器 / Docker 容器（单一镜像）                          │
  │                                                                      │
  │  ┌──────────────────────────────────────────────────────────────┐   │
  │  │  Express Server :9000                                        │   │
  │  │                                                              │   │
  │  │  /api/*    → API 路由（backend 业务逻辑）                      │   │
  │  │  /assets/* → 静态资源（前端构建产物）                           │   │
  │  │  /*        → SPA fallback → index.html（React Router）        │   │
  │  │                                                              │   │
  │  │  ├─ Routes        → HTTP 路由层（薄路由）                      │   │
  │  │  ├─ Services      → 业务逻辑层（排行/统计/缓存计算）            │   │
  │  │  ├─ Repositories  → 数据访问层（TypeORM 查询封装）             │   │
  │  │  ├─ Entities      → 数据库实体（复用 acore-api 定义）          │   │
  │  │  ├─ Middleware    → 请求日志 / 异常处理 / 响应格式化           │   │
  │  │  └─ Shared        → 阵营/种族/职业工具函数 / 坐骑ID常量         │   │
  │  └──────────────────────────────────────────────────────────────┘   │
  │                                                                      │
  └────────────────────────────┼─────────────────────────────────────────┘
                               │
              ┌────────────────┼────────────────┐
              ▼                ▼                ▼
       ┌──────────┐    ┌──────────┐    ┌──────────────┐
       │  Redis   │    │  MySQL   │    │  mount IDs   │
       │  (缓存)   │    │  (AzerothCore)│    │  (内联常量)   │
       └──────────┘    └──────────┘    └──────────────┘
                          │
              ┌───────────┼───────────┐
              ▼           ▼           ▼
       acore_auth   acore_characters  acore_world
```

### 3.2 数据流

```
用户请求 (PC / 移动端 / iframe)
    │
    ▼
┌─────────────────────────────────────┐
│  前端路由匹配对应页面                 │
│  页面组件加载时发起 API 请求          │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  Express Middleware 层               │
│  - API Key 鉴权 (中间件)              │
│  - 请求日志记录 (中间件)              │
│  - 异常统一处理 (中间件)              │
│  - 响应格式化 (中间件)                │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  Service 层                         │
│  1. 尝试从 Redis 读取缓存数据         │
│  2. 缓存命中 → 直接返回               │
│  3. 缓存未命中 → 查询 MySQL           │
│  4. 写入 Redis 缓存并返回             │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  Repository 层 (TypeORM)            │
│  执行 SQL 查询（多数据库连接）         │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│  响应数据格式化（阵营/种族/职业转换）   │
│  JSON 响应 → 前端渲染                 │
└─────────────────────────────────────┘
```

## 五、环境变量配置

```bash
# .env.example

# 应用配置
NODE_ENV=production
PORT=9000
API_KEY=your_api_key_here
LOG_LEVEL=info

# 数据库配置（AzerothCore）
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=acore
DB_PASS=acore
DB_AUTH=acore_auth
DB_CHARACTERS=acore_characters
DB_WORLD=acore_world

# Redis 配置
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_PASSWORD=
REDIS_EXPIRE_TIME=300

# CORS / iframe 配置
ALLOWED_ORIGINS=https://your-wordpress-site.com

# 外部链接
SITE_URL=http://localhost/web-api/
ARMORY_BASE_URL=http://lokta.cn:48733
SERVER_NAME=阿拉希
```
