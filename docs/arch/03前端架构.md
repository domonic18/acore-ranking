## 三、核心样式系统

### 3.1 设计原则

- **iframe 优先**：所有页面在 iframe 中渲染，容器宽度不确定（300px ~ 1200px），布局必须自适应
- **移动端优先**：使用 Tailwind 的 mobile-first 断点，小屏幕为默认样式
- **暗色主题**：默认 dark 模式，与魔兽世界 UI 风格一致
- **零自定义 CSS**：优先使用 Tailwind utility classes，仅在必要时使用 CSS 变量

**模式 A：横向滚动表格**（适用于数据列数 5~7 列）

## 四、API 客户端层

### 4.1 HTTP 客户端封装

```typescript
// src/shared/api/client.ts

const API_BASE_URL = import.meta.env.VITE_API_BASE_URL || '';
const API_KEY = import.meta.env.VITE_API_KEY || '';

function buildUrl(path: string): string {
  // 生产环境（单体部署）：API_BASE_URL 为空，使用相对路径
  // 开发环境：API_BASE_URL = http://localhost:9000
  const base = API_BASE_URL.replace(/\/$/, '');
  const url = base ? `${base}${path}` : path;
  return url;
}

async function request<T>(path: string, options?: RequestInit): Promise<T> {
  const url = new URL(buildUrl(path), window.location.origin);
  if (API_KEY) {
    url.searchParams.set('_key', API_KEY);
  }

  const response = await fetch(url.toString(), {
    headers: {
      'Content-Type': 'application/json',
      ...options?.headers,
    },
    ...options,
  });

  if (!response.ok) {
    throw new Error(`HTTP ${response.status}: ${response.statusText}`);
  }

  const result = await response.json();
  if (!result.success) {
    throw new Error(result.error || 'API request failed');
  }

  return result.data;
}

export const apiClient = {
  get: <T>(path: string) => request<T>(path, { method: 'GET' }),
};
```

### 4.2 单体部署下的 API 调用

生产环境（单体部署）下，`VITE_API_BASE_URL` 为空字符串，API 请求使用**相对路径**：

```

     │
     └── 请求 /api/ranking/gold
         │
         └── Express 同容器处理 → 无需跨域
```

开发环境通过 `.env.development` 配置完整地址：

```bash

```

## 七、组件开发规范

### 7.1 组件组织原则

| 类型 | 位置 | 说明 |
|------|------|------|
| **页面组件** | `src/pages/` | 路由入口，仅组合 feature 组件 |
| **Feature 组件** | `src/features/X/components/` | 特定业务领域的组件 |
| **共享组件** | `src/shared/components/` | 跨 feature 复用 |
| **shadcn/ui 组件** | `src/shared/components/ui/` | CLI 生成的基础组件 |

### 7.2 组件文件命名

- 组件文件使用 PascalCase：`OnlinePlayerTable.tsx`
- Hook 文件使用 camelCase 前缀 `use`：`useOnline.ts`
- API 文件以 `.api.ts` 后缀：`online.api.ts`
- 类型文件以 `.types.ts` 后缀：`online.types.ts`

### 7.3 组件示例：通用表格封装

```tsx
// src/shared/components/DataTable.tsx

import {
  useReactTable,
  getCoreRowModel,
  flexRender,
  type ColumnDef,
} from '@tanstack/react-table';
import {
  Table,
  TableBody,
  TableCell,
  TableHead,
  TableHeader,
  TableRow,
} from '@/shared/components/ui/table';

interface DataTableProps<TData> {
  data: TData[];
  columns: ColumnDef<TData>[];
  loading?: boolean;
}

export function DataTable<TData>({ data, columns, loading }: DataTableProps<TData>) {
  const table = useReactTable({
    data,
    columns,
    getCoreRowModel: getCoreRowModel(),
  });

  return (
    <div className="overflow-x-auto rounded-md border">
      <Table className="min-w-[600px]">
        <TableHeader>
          {table.getHeaderGroups().map(headerGroup => (
            <TableRow key={headerGroup.id}>
              {headerGroup.headers.map(header => (
                <TableHead key={header.id}>
                  {flexRender(header.column.columnDef.header, header.getContext())}
                </TableHead>
              ))}
            </TableRow>
          ))}
        </TableHeader>
        <TableBody>
          {loading ? (
            <TableRow>
              <TableCell colSpan={columns.length} className="h-24 text-center">
                数据加载中...
              </TableCell>
            </TableRow>
          ) : (
            table.getRowModel().rows.map(row => (
              <TableRow key={row.id}>
                {row.getVisibleCells().map(cell => (
                  <TableCell key={cell.id}>
                    {flexRender(cell.column.columnDef.cell, cell.getContext())}
                  </TableCell>
                ))}
              </TableRow>
            ))
          )}
        </TableBody>
      </Table>
    </div>
  );
}
```

## 八、Vite 构建配置

### 8.0 单体部署说明

前后端构建为**单一 Docker 镜像**，前端产物由 Express 静态文件中间件托管。生产环境下：

- 前端无需独立的服务器或域名
- Vite 构建产物（`frontend/dist/`）在 Docker 构建阶段被复制到 `backend/dist/public/`
- Express `express.static()` 托管所有静态资源，`/*` fallback 到 `index.html` 支持 SPA 路由
- 前端 API 调用使用**相对路径**（同域），无需配置 `VITE_API_BASE_URL`

详见 [04部署架构.md](./04部署架构.md)。

### 8.1 基础配置

```typescript
// vite.config.ts

import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import path from 'path';

export default defineConfig({
  plugins: [react()],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
    },
  },
  build: {
    outDir: 'dist',               // 默认输出到 frontend/dist/
    emptyOutDir: true,            // 构建前清空
    sourcemap: process.env.NODE_ENV !== 'production',
    rollupOptions: {
      output: {
        // 代码分割策略
        manualChunks: {
          // React 核心库单独打包
          'vendor-react': ['react', 'react-dom', 'react-router-dom'],
          // 数据获取和表格库
          'vendor-data': ['@tanstack/react-query', '@tanstack/react-table'],
          // UI 组件库
          'vendor-ui': ['lucide-react'],
        },
      },
    },
  },
  // 开发服务器代理（开发时转发 API 请求到后端）
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:9000',
        changeOrigin: true,
      },
    },
  },
});
```

### 8.2 代码分割效果

| Chunk | 内容 | 预估体积 |
|-------|------|---------|
| `vendor-react` | react + react-dom + router | ~130KB gzipped |
| `vendor-data` | query + table | ~45KB gzipped |
| `vendor-ui` | lucide icons | ~20KB gzipped |
| `page-ranking` | 排行页面（懒加载）| ~15KB gzipped |
| `page-hardcore` | 硬核页面（懒加载）| ~12KB gzipped |
| 其他页面 | 各自独立 chunk | ~10KB each |

## 九、iframe 嵌入适配

### 9.1 页面级 iframe 适配

所有页面默认支持 iframe 嵌入，无需额外布局容器：

```tsx
// src/app/App.tsx

import { RouterProvider } from 'react-router-dom';
import { router } from './router';

export default function App() {
  return (
    // 无固定宽度限制，容器宽度由 iframe 或浏览器决定
    <div className="min-h-screen bg-background text-foreground">
      <RouterProvider router={router} />
    </div>
  );
}
```

### 9.2 页面内容最小宽度

每个页面组件内部控制最小宽度，防止在过窄 iframe 中布局崩坏：

```tsx
// src/pages/RankingPage.tsx

export default function RankingPage() {
  return (
    <main className="mx-auto w-full min-w-[320px] max-w-6xl p-4">
      <RankingTabs />
    </main>
  );
}
```

### 9.3 嵌入端建议

```html
<!-- 最小推荐宽度 320px，高度根据内容调整 -->

  src="https://acore-ranking.example.com/ranking"

  style="min-width: 320px; border: none;"
```
