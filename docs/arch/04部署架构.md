# acore-ranking 部署架构设计

## 一、部署模式

### 1.1 单体容器部署

前后端构建为**单一 Docker 镜像**，由 Express 统一托管：

| 层级 | 处理方式 | 说明 |
|------|---------|------|
| **前端** | Vite 构建产物 → 静态文件 | 由 Express `express.static()` 托管 |
| **后端** | Express API → `/api/*` 路由 | 业务逻辑、数据库查询、缓存 |
| **SPA Fallback** | `/*` → `index.html` | 支持 React Router BrowserHistory |

**优势**：
- 腾讯云 SCF 只需部署**一个云函数**，一个容器实例
- 减少网络延迟（前端→后端为同容器内部调用）
- 简化运维（单镜像、单容器、单日志流）
- 避免前后端分离部署的 CORS 和跨域配置复杂度

## 三、统一 Dockerfile

```dockerfile
# docker/Dockerfile
# 前后端统一构建，输出单一可运行镜像

# ============================================================
# Stage 1: 前端构建
# ============================================================
FROM node:20-alpine AS frontend-builder
WORKDIR /app/frontend

COPY frontend/package*.json ./
RUN npm ci

COPY frontend/ ./
RUN npm run build
# 输出: /app/frontend/dist/ (HTML/CSS/JS/静态资源)

# ============================================================
# Stage 2: 后端构建
# ============================================================
FROM node:20-alpine AS backend-builder
WORKDIR /app/backend

COPY backend/package*.json ./
RUN npm ci

COPY backend/ ./
RUN npm run build
# 输出: /app/backend/dist/ (JS + .d.ts)

# ============================================================
# Stage 3: 生产运行镜像
# ============================================================
FROM node:20-alpine AS production
WORKDIR /app

# 安装生产依赖（仅 backend 的 dependencies）
COPY backend/package*.json ./
RUN npm ci --only=production && npm cache clean --force

# 复制后端构建产物
COPY --from=backend-builder /app/backend/dist ./dist

# 复制前端构建产物到后端静态文件目录
COPY --from=frontend-builder /app/frontend/dist ./dist/public

# 非 root 用户运行（安全最佳实践）
RUN addgroup -g 1001 -S nodejs && \
    adduser -S nodejs -u 1001
USER nodejs

# 腾讯云 SCF Web 容器默认端口
EXPOSE 9000

CMD ["node", "dist/server.js"]
```

### 镜像体积优化

| 阶段 | 体积 | 说明 |
|------|------|------|
| 前端构建 | ~300MB（含 node_modules）| 中间层，不进入最终镜像 |
| 后端构建 | ~400MB（含 node_modules）| 中间层，不进入最终镜像 |
| **生产镜像** | **~120MB** | 仅含运行时 + 编译产物 + 生产依赖 |

## 四、Express 静态文件托管配置

生产环境下，Express 同时承担 API 服务和静态文件托管职责：

```typescript
// backend/src/app.ts

import express, { Application } from 'express';
import path from 'path';

export function createApp(): Application {
  const app = express();

  // 1. 安全中间件
  app.use(helmet({ frameguard: false }));
  app.use(iframeCors);

  // 2. 解析中间件
  app.use(express.json());
  app.use(express.urlencoded({ extended: true }));

  // 3. 日志与格式化
  app.use(requestLogger);
  app.use(responseFormatter);

  // 4. 鉴权
  app.use(apiKeyAuth);

  // 5. API 路由（必须在静态文件之前注册，避免被 SPA fallback 拦截）
  app.use('/api/online', onlineRoutes);
  app.use('/api/ranking', rankingRoutes);
  app.use('/api/hardcore', hardcoreRoutes);
  app.use('/api/achievement', achievementRoutes);
  app.use('/api/banlist', banlistRoutes);
  app.use('/api/character', characterRoutes);
  app.use('/api/health', healthRoutes);

  // 6. 静态文件托管（前端构建产物）
  app.use(express.static(path.join(__dirname, 'public')));

  // 7. SPA fallback：所有非 API 请求返回 index.html
  // 支持 React Router 的 BrowserHistory 模式
  app.get('*', (_req, res) => {
    res.sendFile(path.join(__dirname, 'public', 'index.html'));
  });

  // 8. 全局错误处理（末尾注册）
  app.use(errorHandler);

  return app;
}
```

### 路由优先级说明

```
请求到达顺序：
  /api/online/count    → API 路由处理（第 5 步）
  /assets/icons/xxx.png → 静态文件（第 6 步）
  /ranking             → SPA fallback → index.html（第 7 步）
  /character/xxx       → SPA fallback → index.html（第 7 步）
```

> **注意**：API 路由必须注册在 `express.static()` 之前，否则 `/api/health` 等路径可能被误判为静态文件请求。

## 五、本地开发（Docker Compose）

```yaml
# docker-compose.yml

version: '3.8'

services:
  app:
    build:
      context: .
      dockerfile: docker/Dockerfile
      target: production
    ports:
      - "9000:9000"

      - .env

      - NODE_ENV=production

      - mysql

      MYSQL_DATABASE: acore_characters

      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
    restart: unless-stopped

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    restart: unless-stopped

volumes:
  mysql_data:
```

## 七、CI/CD 流水线

### 7.1 GitHub Actions 工作流

```yaml
# .github/workflows/ci.yml

name: Build and Deploy

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

env:
  NODE_VERSION: '20'
  TCR_REGISTRY: ccr.ccs.tencentyun.com
  IMAGE_NAME: acore-ranking

jobs:
  test:
    name: Test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
          cache: 'npm'
          cache-dependency-path: |
            backend/package-lock.json
            frontend/package-lock.json

      - name: Install Backend Dependencies
        working-directory: backend
        run: npm ci

      - name: Install Frontend Dependencies
        working-directory: frontend
        run: npm ci

      - name: Lint
        run: |
          cd backend && npm run lint
          cd ../frontend && npm run lint

      - name: Build Check
        run: |
          cd backend && npm run build
          cd ../frontend && npm run build

  build-and-push:
    name: Build and Push Image
    needs: test
    if: github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Login to TCR
        uses: docker/login-action@v3
        with:
          registry: ${{ env.TCR_REGISTRY }}
          username: ${{ secrets.TCR_USERNAME }}
          password: ${{ secrets.TCR_PASSWORD }}

      - name: Build and Push
        uses: docker/build-push-action@v5
        with:
          context: .
          file: docker/Dockerfile
          push: true
          tags: |
            ${{ env.TCR_REGISTRY }}/${{ secrets.TCR_NAMESPACE }}/${{ env.IMAGE_NAME }}:latest
            ${{ env.TCR_REGISTRY }}/${{ secrets.TCR_NAMESPACE }}/${{ env.IMAGE_NAME }}:${{ github.sha }}
          cache-from: type=gha
          cache-to: type=gha,mode=max

  deploy:
    name: Deploy to SCF
    needs: build-and-push
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup Serverless
        run: npm install -g serverless

      - name: Deploy
        run: serverless deploy
        env:
          TENCENT_SECRET_ID: ${{ secrets.TENCENT_SECRET_ID }}
          TENCENT_SECRET_KEY: ${{ secrets.TENCENT_SECRET_KEY }}
          TCR_NAMESPACE: ${{ secrets.TCR_NAMESPACE }}
          IMAGE_TAG: ${{ github.sha }}
          # 其他环境变量通过 secrets 注入
          DB_HOST: ${{ secrets.DB_HOST }}
          DB_PORT: ${{ secrets.DB_PORT }}
          DB_USER: ${{ secrets.DB_USER }}
          DB_PASS: ${{ secrets.DB_PASS }}
          DB_AUTH: ${{ secrets.DB_AUTH }}
          DB_CHARACTERS: ${{ secrets.DB_CHARACTERS }}
          DB_WORLD: ${{ secrets.DB_WORLD }}
          REDIS_HOST: ${{ secrets.REDIS_HOST }}
          REDIS_PORT: ${{ secrets.REDIS_PORT }}
          API_KEY: ${{ secrets.API_KEY }}
          ALLOWED_ORIGINS: ${{ secrets.ALLOWED_ORIGINS }}
```

## 十、前端构建产物路径说明

### Vite 配置调整

前端构建输出目录使用默认 `frontend/dist`，在 Dockerfile Stage 3 中被复制到后端产物目录：

```typescript
// frontend/vite.config.ts

export default defineConfig({
  // ... 其他配置
  build: {
    outDir: 'dist',           // 默认输出目录
    emptyOutDir: true,        // 构建前清空
    // 前端产物最终被复制到 backend/dist/public/
  },
});
```

### 生产环境 API 调用

由于前后端同域部署，前端 API 调用无需指定完整域名：

```typescript
// src/shared/api/client.ts

const API_BASE_URL = import.meta.env.VITE_API_BASE_URL || '';
// 生产环境: VITE_API_BASE_URL = ''（空字符串）
// 请求 /api/online/count 会被同域 Express 处理
```

开发环境可通过 `.env.development` 配置代理：

```bash
# frontend/.env.development
VITE_API_BASE_URL=http://localhost:9000
```
